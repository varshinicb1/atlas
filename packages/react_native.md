    symptom: Metro bundler fails with resolution errors or transform errors on startup.
    cause: Missing dependencies, incompatible package versions, or cache corruption.
    fix: Delete node_modules and reinstall. Clear Metro cache (npx react-native start --reset-cache). Check package.json for React Native version compatibility. Verify all native modules have corresponding pod install (iOS) or gradle sync (Android).
  - id: failure:react-native/fast-refresh-not-working
    symptom: Code changes do not reflect in the running app — requires full rebuild.
    cause: Fast Refresh conflicts with certain patterns (redux-like global state, class components, inline require, or broken error boundaries).
    fix: Use functional components with hooks. Avoid inline requires. Add error boundaries to catch Fast Refresh recovery errors. Check Metro terminal for Fast Refresh status messages.
  - id: failure:react-native/hermes-crashes
    symptom: App crashes on startup or during specific operations with Hermes enabled.
    cause: Hermes incompatibility with certain JavaScript features or native module assumptions.
    fix: "Disable Hermes temporarily (set enableHermes: false in gradle.properties and Podfile) to confirm Hermes is the cause. Check for unsupported features (Proxy, eval, Symbol.toStringTag on some APIs). Update React Native version for Hermes fixes."
  - id: failure:react-native/keyboard-avoidance
    symptom: Content is hidden behind the keyboard when TextInput is focused.
    cause: No keyboard-aware scrolling — the keyboard overlays the input without adjusting the view.
    fix: Use KeyboardAvoidingView with behavior='padding' (iOS) or 'height' (Android). Wrap scrollable content in ScrollView with keyboardShouldPersistTaps='handled'. Use react-native-keyboard-aware-scroll-view for more control.

React Native's core innovation is the JavaScript-to-Native bridge (now evolving into JSI). Instead of running a WebView, React Native communicates with native UI widgets through a serialized bridge (JSON messages) or direct C++ function calls (JSI + JSI). This means React Native apps render with the same native components as Swift/Kotlin apps — UIKit on iOS, Android Views on Android — while the business logic runs in JavaScript.

The New Architecture (Fabric + TurboModules + JSI) represents a fundamental re-architecture of React Native. Fabric enables synchronous layout calculations, better interop with React 18's concurrent features, and cached props that reduce bridge traffic. TurboModules lazily load native modules — instead of loading all native modules at startup, each module loads on first use, reducing initial memory and startup time by up to 50%.

Expo has become the recommended way to start React Native projects. It provides 200+ pre-built native modules, over-the-air updates (expo-updates), cloud builds (EAS Build), and a managed workflow that abstracts Xcode and Android Studio configuration. The expo-dev-client bridges managed and bare workflows — use Expo's modules while retaining the ability to add custom native code when needed.
